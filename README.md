# bigdata-subway-station-analysis

# 신규 지하철역 필요 지역 분석 시스템 설계

## 1. 프로젝트 개요 및 문제 정의
현재 수도권의 대중교통망은 잘 구축되어 있지만, 실제 생활인구나 유동인구에 비해 지하철 접근성이 부족한 지역이 존재한다. 예를 들어 명지대학교 주변처럼 버스 의존도가 높고 출퇴근 시간 혼잡이 심한 지역들이 있다.
본 프로젝트에서는 생활인구 데이터와 버스·지하철 이용 데이터를 활용하여 교통 수요가 높지만 상대적으로 교통 인프라가 부족한 지역을 분석하고, 신규 지하철역이 필요한 후보 지역을 찾아보고자 한다.

## 2. 사용 데이터
본 프로젝트에서는 다음과 같은 공공데이터를 활용한다.
* 서울교통공사_지하철혼잡도정보
* 한국도로공사_혼잡빈도
* 한국관광공사_관광빅데이터 정보서비스_GW
* 서울특별시_지하철 호선별 역별 시간대별 승하차 인원 정보
* 서울특별시_버스노선별 정류장별 승하차 인원 정보
* 서울특별시_자치구단위 서울생활인구 일별 집계표

## 3. 사용 기술 및 환경
* Python
* Hadoop HDFS
* Apache Spark
* Apache Hive
* Matplotlib
* Folium
수집한 데이터를 Hadoop 환경에 저장하고, Spark를 이용해 전처리 및 분석을 진행한다. 이후 분석 결과를 지도 기반으로 시각화하여 교통 취약 지역을 확인한다.
